# AI Medication Management & Chatbot Backend

## What This Is

클로바 OCR을 활용하여 처방전 이미지에서 약품명을 추출 및 자체 데이터베이스와 매칭하고, 추출된 정보를 바탕으로 GPT 기반의 챗봇('약속이 상담소')과 연동하여 환자에게 정확한 약품 정보와 맞춤형 상담을 제공하는 웹/앱 백엔드 시스템입니다.

## Core Value

환자에게 정확한 약품 정보를 신속하게 제공하고, 분석된 처방전 맥락을 바탕으로 한 맞춤형 챗봇 상담을 제공하여 약물 복용의 안전성과 편의성을 동시에 극대화한다.

## Requirements

### Validated

- ✓ 기존 인증/유저 기반 시스템 동작
- ✓ 시딩 스크립트 연결 완료 (이메일로 전달받은 eyak_data_raw.csv 총 4711건 데이터 연동)

### Active

- [ ] [OCR] 데이터 조회: 매칭된 약품의 기본 정보를 DB에서 조회
- [ ] [OCR] AI 보완: DB에 없는 정보 또는 추가 설명이 필요한 경우 GPT 보완
- [ ] [OCR] 실패 처리: OCR 또는 AI 분석 실패 시 케이스별(ocr 실패, gpt 연동 실패 등) 명확한 메시지 분리
- [ ] [Chatbot] 세션 관리: 챗봇 진입 시 세션 생성, 시작된 뷰의 맥락(처방전/리포트/복약기록) 연결 및 저장
- [ ] [Chatbot] 히스토리 관리: 대화 기록의 소프트 딜리트(Soft Delete) 기능 
- [ ] [Chatbot] 답변 연동: 과거 분석된 OCR 결과를 컨텍스트로 전달하여 사용자 질문에 응답
- [ ] [Chatbot] FAQ 로직: 고정된 질문 추가 및 AI API 호출 없이 즉시 답변하여 호출 비용 절감 (최우선 순위가 아님)

### Out of Scope

- 음성 텍스트 변환(STT/TTS) 연동 챗봇 앱 개발
- 직접적인 LLM 자연어 딥러닝 모델 학습 (시간 비용 대비 비효율적이므로 외부 리소스 활용)

## Context

- 현재 OCR의 약품 텍스트 매칭 시딩 기능은 이미 연동 완료된 상태입니다.
- 프로젝트 핵심은 OCR 매칭된 데이터를 '약속이 상담소(챗봇)'와 자연스럽게 연결하고, 세션 문맥을 기억하여 개인화된 경험을 제공하는 데 있습니다.

## Constraints

- **성능/비용 최적화**: 챗봇 처리 시 단순 답이 정해진(FAQ) 질문은 값비싼 GPT 호출 로직을 스킵해야 함.
- **데이터 보존 제약**: 사용자가 채팅을 삭제하길 요청해도 DB에서 완전 소멸시키지 않고 소프트 딜리트로 관리할 것.
